# 订单调度算法三阶段优化开发任务清单

## 阶段一：路线方向角一致性优化（防止“南辕北辙”）
目前的调度策略已经支持一辆车处理多个订单，但缺乏对后续订单所导致路线偏离的考量。为了减少无意义的额外行驶距离，需要引入路线方向角的对比机制。
- [x] **方向角计算逻辑实现**：开发基于经纬度坐标的向量计算工具函数，计算当前路径行驶总体方向矢量与候选任务（起点到终点）方向矢量之间的夹角。
- [x] **偏航惩罚机制（Directional Penalty）设计**：在现有的 `Cost` 函数评估中加入方向角惩罚项。如果夹角过大（如出现钝角甚至相反方向），大幅增加其Cost值。
- [x] **车辆作业路线平滑度预校验**：评估连续接单带来的轨迹规划变化，尽量保持车辆在一个大致象限或方向延伸内作业，减少折返跑情况。

## 阶段二：模拟退火算法替换贪心算法
将现有的“基于cost函数计算值的贪心分配”改造为具备跳出局部最优能力的模拟退火算法（Simulated Annealing, SA）。
- [x] **SA算法基础框架搭建**：建立模拟退火的循环迭代框架，设置初始温度（Initial Temperature）、冷却系数（Cooling Rate）和终止降温条件。
- [x] **邻域搜索方案（Neighborhood Search）设计与实现**：
    - [x] **Swap（交换）算子**：随机选择两辆不同的可执行任务车辆，交换它们各自负责的一个或多个任务。
    - [x] **Move（移动）算子**：随机选择一个任务，将其从当前负责的货车转移指派给另一辆货车。
- [x] **状态评估与转移逻辑（Metropolis准则）**：计算执行算子后全局系统的新总代价（Total Cost）。若更优则接受；若更差，则以一定概率接受（基于当前温度控制），避免陷入局部最优。
- [x] **算法性能调优**：结合历史订单Mock数据，测试算法寻找最优解的执行耗时与分配结果，微调退火参数以实现最优平衡。

## 阶段三：基于LIFO装卸逻辑的“一笔画”绕路代价控制
在多单拼载的情况下，“一笔画”的总代价不能仅仅考虑距离，还必须结合货车后备箱本身的物理约束（后进先出，即栈结构）及体积-代价比。
- [x] **装卸货形态栈模拟（LIFO逻辑校验）**：
    - [x] 在车辆路线规划及任务插入阶段，增加对车厢货物排序的合法性验证：所有货物看作一个「栈」。先装进去的货必定在车厢最里侧，必须最后卸下。
    - [x] 校验逻辑：任何拼单路线在模拟装卸车时，如果要卸下目标由于被其他较晚装上的货物挡住而违背了LIFO原则，则该分配组合视为无效（或惩罚Cost设为极大值）。
- [x] **边际绕路惩罚机制（防止捡芝麻丢西瓜）**：
    - [x] 引入“边际载运性价比”概念。当系统尝试将一个小订单（例如几百公斤的零散货物）分配至该车厢时，由于该任务需要额外去新的A点装车并在B点卸车，计算此次插入操作导致的净增行驶距离（Detour Distance）。
    - [x] 如果增量距离与订单重量/运费比值突破合理阈值（即为了小单绕远路），产生巨额惩罚Cost，使算法倾向于放弃拼加该微小订单，将其交由其他顺路车辆或更合适的车辆执行。
- [x] **序列重排的合法性保护**：在阶段二的SA算法中，所有产生的 Swap 与 Move 动作必须经过上述栈逻辑合法性与绕路代价的前置校验后才能真正参与下一次的温度迭代。
